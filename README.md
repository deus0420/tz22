### Анализ процесса и выявленные противоречия
На основе описания "как есть" были выявлены следующие проблемы:

1. **Избыточность систем:** Использование трех независимых систем (№1, №2, №3) приводит к двойной работе. Техподдержка вынуждена закрывать одну и ту же заявку дважды вручную.
2. **Отсутствие контроля:** В описании отсутствует этап согласования закупки. Снабжение заказывает оборудование у поставщика без подтверждения бюджета.
3. **Информационный разрыв:** Пользователь видит номер обращения в Системе №1, но процесс закупки происходит в Системе №2 и №3. При отсутствии интеграции пользователь не получает актуальный статус "Ждем поставку".
4. **Непрозрачность логистики:** Вызов логистики описан как ручное действие, которое не фиксируется в системах, что может привести к потере оборудования при передаче.
